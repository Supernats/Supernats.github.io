I'm a web developer in the San Francisco Bay Area. I build apps in Ruby on Rails and Javascript, with a strong love of Backbone.

Here are some recent projects I've worked on

Paytoma | Social bill splitting and loan tracking app. Backbone.js running on RESTful Rails API. Tracks data for unregistered users through single table inheritance. Automatic dynamic splitting of transactions between multiple parties. Source. Live.

Levithan | Browser version of the Nokia classic. Built in Javascript, using jQuery for Direct DOM manipulation. Source.

Chess | Pure Ruby CLI board game. Strong object orientation and inheritance. Source.

Omniloader | Ruby script for automated, single-step setup of a vanilla terminal to a dev's favored Vim and bash settings. Uninstall reverts the machine back to its initial setup like nothing was changed. Source.

ActiveRecord Lite | Homebrewed clone of Ruby's ActiveRecord ORM. Whitelisting, mass assignment, associations, querying, database persistence. Source.

Towers of Hanoi | Basic jQuery and super 90s CSS. It gives me an unreasonable degree of joy. Live. Source.

Tic Tac Toe | Companion piece to Hanoi. Ridiculosity runs high. Live. Source.

I attack every situation I face with the same mentality; if it's worth doing, it's worth doing right. It's all well and good to make something that "works," but I need to get into it and exploit all the "hows" of that functionality.

Object orientation is my heart and method decomposition is the blood it pumps. In the battle of concision vs expression, I will always come to bat for expression. Top level execution should always be as abstracted as possible; if something can be represented as one atomic piece of functionality, it must.

If this sounds like the kind of thing you're interested in, I'd love to talk. Thanks for stopping by.
